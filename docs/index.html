Multi-Environment Crop Analytics

Bridging Phenomics and Climatic Big Data

MERIDIAN is a premium scientific computing suite engineered to dissect genotype-by-environment (G×E) interactions, correct spatial field trials, partition linear mixed model variance, and characterize adaptive crop traits using stage-aware agroclimatic covariates.

Functional Architecture

Unified Scientific Modules

Dissect phenotypic variation, model spatial field trends, conduct multi-environment variance partitioning, and compute stage-specific climate responses in one cohesive workspace.

1. Unified Data Engine

Seamlessly upload, clean, and validate massive multi-environment trials (MET) in CSV and Excel formats. Features automatic robust date parser, missing value imputation, and field trial mapping.

2. Interactive EDA

Explore trait distributions across environments using glassmorphic density plots, interactive scatter matrix grids, Pearson correlation maps, and automated boxplot outlier identification.

3. Spatial Correction

Remove experimental row-and-column trends using advanced spatial mixed modeling (SpATS or custom spatial algorithms). Automatically fit 2D tensor-product P-splines to reveal true genetic merits.

4. ANOVA & Mixed Models

Partition overall phenotypic variation via Linear Mixed Models (LMM) with REML. Calculate broad-sense heritability ($H^2$), extract genetic BLUPs and BLUEs, and model G×E structures.

5. Stability & GxE Models

Dissect environmental adaptability using rigorous statistical frameworks: Finlay-Wilkinson regressions, Additive Main Effects and Multiplicative Interactions (AMMI) models, and GGE biplots.

6. Enviromic Integration

Query NASA POWER daily climate databases in real time. Divide growth cycles dynamically into Vegetative, Reproductive, and Late stages to compute stage-aware indices (e.g. GDD, thermal stress days).

Scientific Excellence

Publication-Grade Biometrics

MERIDIAN implements rigorous mathematical equations mapped to major peer-reviewed breeding journals. We represent mixed model structures and multi-environment interactions accurately.

We leverage advanced tensor-product P-splines to partition localized field row-column gradients cleanly, bypassing experimental design limitations seamlessly.

By integrating AMMI Singular Value Decomposition, the system isolates genotype variance components across PCA projections to locate optimal breeding targets.

1. Linear Mixed Model (LMM) Structure
\[ \mathbf{y} = \mathbf{X}\boldsymbol{\beta} + \mathbf{Z}\mathbf{u} + \boldsymbol{\varepsilon} \]

Where \(\mathbf{y}\) represents phenotypic measurements, \(\boldsymbol{\beta}\) is the vector of environmental and block fixed effects, \(\mathbf{u} \sim \mathcal{N}(\mathbf{0}, \mathbf{G})\) is the vector of random genotypic effects, and \(\boldsymbol{\varepsilon} \sim \mathcal{N}(\mathbf{0}, \mathbf{R})\) represents standard experimental error components.

2. Additive Main Effects & Multiplicative Interaction
\[ Y_{ij} = \mu + g_i + e_j + \sum_{k=1}^{n} \lambda_k \alpha_{ik} \gamma_{jk} + \theta_{ij} \]

Where \(Y_{ij}\) is the mean trait value of genotype \(i\) in environment \(j\), \(g_i\) and \(e_j\) represent main genotype and environmental effects, \(\lambda_k\) represents eigenvalues of PC axis \(k\), and \(\alpha_{ik}, \gamma_{jk}\) are genotypic and environmental eigenvector loadings.

Quick Deployment

Deploy MERIDIAN to Your Machine

Run standard package commands directly in R or RStudio to boot the full application environment locally.

R CONSOLE INSTALLATION
# 1. Install devtools package if missing
if (!require("devtools", quietly = TRUE)) install.packages("devtools")

# 2. Retrieve MERIDIAN directly from your repository
devtools::install_github("SalvaOsuna/MERIDIAN")

# 3. Initialize the application server
library(MERIDIAN)
run_app()
